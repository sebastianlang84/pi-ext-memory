# docs/README

## Boundary

- Root living docs stay at repo root when they are primary entry points (`README.md`, `TODO.md`, `CHANGELOG.md`, `AGENTS.md`).
- Product intent, scope, requirements, risks, and success criteria belong in `docs/product/`.
- User-facing commands, tools, workflows, configuration, and troubleshooting belong in `docs/user/`.
- Developer-facing architecture, module boundaries, schemas, APIs, tests, and local development notes belong in `docs/developer/`.
- Durable decisions belong in `docs/adr/`.
- Work breakdowns belong in `docs/plans/`.
- Superseded drafts, snapshots, or historical material belong in `docs/archive/`.
